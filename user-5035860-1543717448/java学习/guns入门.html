####guns文件夹导入

  • 点击eclipse菜单File->import->Maven-> Existing maven project
  • 找到guns项目目录，并点击所有模块，之后点击Finish，导入成功
    ####使用eclipse运行
    *先运行guns-admin\src\main\java\com\stylefeng\guns\GunsApplication.java
  • 出现下列语句则运行成功
  • 在浏览器中输入http://localhost:8080/guns即可打开后台登陆界面

PS：默认登录账号密码:admin/111111

####Guns两种打包方式
jar包

  • 打包之前修改 guns-admin中的pom.xml中的packaging 节点，改为jar

  • 在项目的guns-parent目录执行 clean package -Dmaven.test.skip=true

  • 命令执行成功后，在 guns-admin/target 目录下即可看到打包好的文件
  • 在cmd中运行java -jar guns-admin-1.0.0.jar(运行这个之前tomcat不能运行)
  • 在浏览器中输入http://localhost:8080/guns即可打开后台登陆界面

war包

  • 打包之前修改 guns-admin中的pom.xml中的packaging 节点，改为war
  • 打包方式与jar包一样
  • 生成的war包拉出来改名为gun.war,然后直接运行就行
    ####文件结构
  • guns-admin 模块为后台管理系统模块，包括管理系统的业务代码，前端页面，项目的配置信息等等
  • guns-core 模块为抽象出的核心（通用）模块，以供其他模块调用，此模块主要封装了一些通用的工具类，公共枚举，常量，配置等等
  • guns-generator 为代码生成模块，其中代码生成模块整合了mybatis-plus的代码生成器和guns独有的代码生成器，可以一键生成entity，dao，service，html，js等代码，可减少新业务 70% 的工作量
  • guns-parent 模块为其他所有模块的父模块，主要功能是管理项目中所有用到的jar，以及聚合其他模块
  • guns-rest 为专门提供restful api的模块，该模块中主要实现了jwt鉴权和传输数据签名的机制
